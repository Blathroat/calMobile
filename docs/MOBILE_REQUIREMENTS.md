# 移动端需求文档

## 1. 项目目标

开发一个移动端展会排期与报名应用。本文档由根目录 `TODO.md` 的浏览器端需求改写而来，保留原有业务范围，并将鼠标悬停、桌面表格、网页弹窗等交互转换为 Android 触控端可执行的页面与操作。

当前已确认约束：原生 Android 应用，使用 Java 构建，目标使用 API 30。除这些约束外，后端、数据库、认证、富文本与媒体存储等重要技术选型尚未确认，不在本阶段擅自决定。

## 2. 角色与权限

应用暂设四类权限身份，且同一账号可同时持有多种身份：

- 超级管理员：拥有全部权限，并可进入系统设置。
- 普通管理员：管理所有展会与用户，可锁定展会、调整展会状态、处理用户状态。
- 展商：发布、修改、删除自己的展会，进入展商后台，管理自己展会的报名。
- 普通用户：查看展会、查看展商或用户主页、报名展会、管理自己的报名。

用户账号需要保留账号状态字段，用于管理员封禁、解封或限制违规用户。

## 3. 移动端信息架构

建议移动端采用单 Activity 或多 Activity 均可承载的页面结构，具体导航框架待确认后再定。需求层面先定义以下入口：

- 首页：展会月视图与日期展会列表。
- 我的报名：当前用户已报名展会列表。
- 个人主页：个人资料与账号设置入口。
- 展商后台：仅展商可见。
- 管理后台：仅管理员或超级管理员可见。

所有页面顶部保留应用标题区域。未登录时展示注册/登录入口；已登录时展示用户名入口，点击进入个人主页。

## 4. 移动端交互适配

浏览器端需求中的交互在移动端按以下规则改写：

- 鼠标悬停展示详情：改为点击日期、展会卡片或信息图标后展示底部弹层或详情页。
- 桌面表格：改为卡片列表，关键字段直接展示，次要字段进入详情页。
- 页面内弹窗：改为底部弹层或全屏表单页，避免小屏遮挡输入。
- 滚轮翻页：改为垂直滚动列表；超过 5 个展会时列表可滚动。
- 过长文本悬停展示：改为点击字段展开、进入详情页或显示只读弹层。
- 状态圆角矩形：保留为移动端状态标签，用不同颜色突出展会状态。

## 5. 页面需求

### 5.1 首页与月视图

首页以月视图为核心，展示当前月份日期。存在展会的日期需要有简易标记。用户点击某一天后，展示当天展会列表；每个列表项展示展会名称、展商名称、地点、日期时间与状态标签，并提供查看详情入口。

若某一天展会超过 5 个，列表区域使用垂直滚动展示，不挤压月视图主体。

### 5.2 展会详情

展会详情页展示展会名称、展商名称、地点、日期时间、状态、展会介绍，以及展商上传的图片或视频内容。报名按钮根据权限和展会设置展示不同状态：

- 可报名：按钮可点击，进入报名信息填写流程。
- 无权限：按钮置灰并说明无报名权限。
- 仅展示：按钮置灰并显示“展会仅作为展示”。
- 管理员暂停或报名截止：按钮置灰并显示对应原因。

点击展商名称进入展商公开主页。

### 5.3 报名流程

报名信息由展商配置。字段类型包含单选、多选、填空三种；展商也可以设置不收集额外信息。若需要审核，用户提交或修改报名后进入待审核状态；若不需要审核，提交后直接生效。

报名表单在移动端优先使用全屏表单页或底部弹层。表单提交前需要展示当前展会名称与报名方式，避免用户误提交。

### 5.4 我的报名

“我的报名”页面展示当前用户已报名的展会卡片列表。每个卡片展示展会名称、展商名称、地点、展会时间、审核状态，并提供撤销或修改报名入口。修改后若该展会需要审核，则修改记录重新进入展商审核流程。

### 5.5 个人主页与账号设置

个人主页展示头像、昵称、联系方式、社交媒体地址、简介、已报名展会展示设置，并提供账号设置入口。

账号设置页允许修改密码、头像、昵称、简介、联系方式、社交媒体地址，以及是否公开展示已参加展会。公开范围包含：不展示、公开展示、仅向展商展示。若选择公开展示或仅向展商展示，需要额外选择展示多久内的展会。

### 5.6 用户与展商公开主页

所有页面中可点击的用户名称进入该用户公开主页。公开主页展示头像、昵称、简介、联系方式与社交媒体地址。已报名展会列表仅在该用户设置允许时展示。

所有页面中可点击的展商名称进入展商公开主页。展商公开主页展示该展商发布的展会列表，用户可继续查看详情或报名。

### 5.7 展商后台

展商后台提供“我举办的展会”列表。展商可发布、修改或删除自己的展会，可设置报名方式：报名需审核、报名无需审核、仅展示。

当报名方式为需审核或无需审核时，展商可设置报名信息收集项，最多 4 项。展会详情支持上传图片和视频，具体上传限制待超级管理员配置与媒体存储方案确认后落地。

展会状态包含报名中、截止报名、已结束、管理员暂停报名。管理员暂停报名状态只能由管理员设置，展商无权自行解除。

### 5.8 展会报名管理

展商进入某个展会的报名管理页后，看到报名用户卡片列表。每个卡片展示用户昵称、报名时间、该用户报名过该展商展会的次数、审核状态，以及报名信息收集项摘要。

点击用户昵称进入用户公开主页。点击“报名历史”或次数区域展示该用户参加过该展商展会的名称列表，此信息不受用户公开展示开关控制。未审核记录展示通过和拒绝操作；修改后需审核的记录需要特别标注。

### 5.9 管理后台

管理员用户管理页展示所有用户列表，包含用户信息、用户状态、用户权限、注册时间、最近登录时间，并可执行封禁、解封、修改权限、修改用户信息、修改用户状态等操作。

管理员展会管理页展示所有展会列表，包含展会信息、展会状态、发布时间、持续时间，并可执行锁定、解锁、修改状态、修改信息、删除等操作。

超级管理员系统设置页可配置每篇富文本图片和视频的最大数量与最大大小。

## 6. 数据对象草案

以下仅为需求层对象草案，不代表最终数据库或接口结构：

- User：账号、头像、昵称、简介、联系方式、社交媒体、权限集合、账号状态、注册时间、最近登录时间、公开展示设置。
- Exhibition：名称、展商、地点、开始时间、结束时间、状态、报名方式、介绍富文本、媒体列表、锁定状态。
- Registration：用户、展会、报名时间、审核状态、是否修改后重新审核、报名表单回答。
- RegistrationField：字段标题、字段类型、选项、是否必填。
- SystemSetting：富文本图片数量限制、图片大小限制、视频数量限制、视频大小限制。

## 7. 当前阶段技术路线

已确认：

- Android 原生项目。
- Java 作为主要开发语言。
- API 30 作为当前编译与目标 API。
- 首阶段先建立可打开的移动端项目骨架和需求文档，不接入真实后端。

暂不擅自决定：

- 后端 API 形态与服务端技术栈。
- 登录认证方案。
- 本地数据库或缓存方案。
- 网络请求库。
- 富文本编辑、渲染与图片/视频存储方案。
- 最低支持 Android 版本。
- 正式应用包名与发布签名。

## 8. 待确认的重要技术选型

进入完整功能开发前，需要确认以下问题：

1. 后端是新建服务端，还是移动端对接已有接口。
2. 登录注册使用自建账号系统、第三方登录，还是两者都支持。
3. 移动端是否需要离线浏览或本地缓存。
4. 富文本内容在移动端只展示，还是也要让展商在移动端编辑。
5. 图片和视频上传存储位置、大小限制和审核策略。
6. 最低支持 Android 版本与正式 applicationId。
7. 管理后台是否必须完整放入手机 App，还是管理员优先使用 Web 后台。

## 9. 验收场景

- Happy path：首页月视图能展示有展会的日期标记，点击日期进入当天展会列表，点击展会进入详情并看到可报名状态。
- Edge path：某一天展会超过 5 个时，展会列表可滚动，月视图布局不被挤乱。
- Regression path：普通用户仍可查看展会详情与报名；展商和管理员入口只在对应权限下出现。
