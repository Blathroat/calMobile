# CalMobile 移动端需求文档

## 1. 项目目标

CalMobile 是一个面向展会排期、展示、报名与管理的 Android 移动端应用。本文档由原浏览器端 `TODO.md` 适配而来，优先覆盖移动端用户体验与业务边界，不预设后端、数据库、认证服务、富媒体存储等重要技术选型。

## 2. 平台约束

- 客户端类型：Android 原生应用。
- 构建语言：Java。
- Android API：以 API 30 为目标版本。
- 当前阶段：先建立移动端需求、项目约定与最小工程骨架；重要技术选型确认后再继续实现完整业务。

## 3. 角色与权限

- 超级管理员：拥有全部管理权限，并可管理系统级富媒体限制。
- 普通管理员：管理全部展会与用户，可锁定/解锁展会、修改展会状态、处理违规用户。
- 展商：发布、修改、删除自己的展会，查看展商后台，管理自己展会的报名记录。
- 普通用户：查看展会、查看详情、提交报名、管理自己的报名记录。
- 同一账号可以持有多种身份；账号有状态字段，用于封禁、限制或恢复。

## 4. 移动端信息架构

### 4.1 首页 / 展会月历

- 首页默认展示月视图排期。
- 有展会的日期显示轻量标记。
- 移动端不使用鼠标悬停；点击日期后以底部弹层或新页面展示该日展会卡片列表。
- 展会卡片展示展会名称、展商名称、地点、日期时间、状态标签和“查看详情”。
- 单日展会超过 5 个时使用可滚动列表，不使用桌面滚轮分页概念。

### 4.2 展会详情

- 展示展会名称、展商名称、地点、日期时间、状态、展会介绍。
- 展会介绍支持富文本及图片/视频展示，具体编辑器、上传和存储方案待确认。
- 展示报名按钮；无权限、仅展示、截止报名、已结束或管理员暂停报名时按钮置灰并说明原因。
- 点击报名后进入报名表单页面或弹层；移动端优先使用独立页面，避免小屏输入拥挤。

### 4.3 报名表单

- 报名信息由展商配置。
- 支持单选、多选、填空三种收集项。
- 展商最多设置 4 个收集项，也可不收集额外信息。
- 报名方式包括需审核、无需审核、仅展示。
- 修改报名后，如该展会需要审核，则状态回到待审核并标注为修改后审核。

### 4.4 我的报名

- 展示当前用户已报名展会列表。
- 移动端使用卡片列表，不使用宽表格。
- 每张卡片展示展会名称、展商、地点、时间、报名状态，并提供撤销或修改入口。

### 4.5 个人主页与账号设置

- 个人主页展示头像、昵称、简介、联系方式、社交媒体地址、已参加展会公开策略。
- 账号设置支持修改密码、头像、昵称、简介、联系方式、社交媒体地址。
- 已参加展会公开策略默认不展示，可选不展示、公开展示、仅向展商展示。
- 若允许展示，则可设置展示多久内的展会。

### 4.6 用户公开页

- 所有用户名可点击进入用户公开页。
- 展示用户头像、昵称、简介、联系方式、社交媒体地址。
- 若用户允许展示，则展示其已报名展会列表。
- 自己访问自己的公开页时，可进入个人主页与账号设置。

### 4.7 展商后台

- 展商可发布、修改或删除自己的展会。
- 可设置报名方式、报名信息收集项、展会详情富媒体内容。
- “我举办的展会”使用移动端列表，每项提供状态设置和报名管理入口。
- 展会状态包括报名中、截止报名、已结束、管理员暂停报名。
- 管理员暂停报名由管理员设置，展商无权自行解除。

### 4.8 展会报名管理

- 展商查看报名用户列表。
- 移动端使用报名记录卡片或分组列表。
- 每条记录展示用户昵称、报名时间、该用户参加过本展商展会次数、报名表单内容、审核状态。
- 长文本内容默认截断，点击后展开详情。
- 未审核记录展示通过和拒绝操作。

### 4.9 管理后台

- 管理员用户管理：查看用户信息、账号状态、权限、注册时间、登录时间，并执行封禁、解封、权限和状态调整。
- 管理员展会管理：查看全部展会信息、状态、发布时间、持续时间，并执行锁定、解锁、状态修改、信息修改和删除。
- 超级管理员系统设置：配置每篇富文本图片/视频最大数量和大小。
- 移动端后台页面优先采用搜索、筛选、卡片详情和分步操作，避免桌面式大表格。

## 5. 核心状态

### 5.1 展会状态

- 报名中：普通用户可报名。
- 截止报名：不可新增报名，可查看详情。
- 已结束：不可报名，可查看详情。
- 管理员暂停报名：不可报名，展商不可自行恢复。
- 锁定：仅管理员和展商本人可查看或修改，普通用户不可见或不可操作。

### 5.2 报名状态

- 待审核。
- 已通过。
- 已拒绝。
- 已撤销。
- 修改后待审核。

## 6. 待确认的重要技术选型

以下内容属于重要技术选型，执行前需要用户确认：

- 是否需要后端服务，以及后端语言/框架。
- 本地数据是否使用 SQLite、Room 或仅临时内存数据。
- 登录认证方式，例如自建账号、第三方登录、短信或邮箱验证。
- 客户端网络层、接口协议与错误处理策略。
- 富文本编辑、图片视频上传、压缩、预览与存储方案。
- 是否需要推送通知、离线缓存、日历系统集成。
- Android UI 技术栈是否保持传统 View/XML，或改用其他方案。

## 8. 验收场景

### 场景 A：普通用户浏览并报名

- 入口：打开应用首页。
- 条件：某日期存在报名中的展会。
- 操作：点击日期，查看展会卡片，进入详情，点击报名，填写表单并提交。
- 通过条件：展会详情展示完整信息，报名按钮可用，提交后产生报名记录。

### 场景 B：不可报名状态处理

- 入口：展会详情页。
- 条件：展会处于仅展示、截止报名、已结束或管理员暂停报名。
- 操作：查看详情页报名区域。
- 通过条件：报名按钮置灰，并显示明确不可报名原因。

### 场景 C：展商审核报名

- 入口：展商后台的报名管理页。
- 条件：存在待审核或修改后待审核记录。
- 操作：查看报名内容，点击通过或拒绝。
- 通过条件：报名记录状态更新，用户列表中能区分原始待审核与修改后审核。

### 场景 D：管理员暂停展会

- 入口：管理员展会管理页。
- 条件：存在报名中的展会。
- 操作：将展会状态设置为管理员暂停报名。
- 通过条件：普通用户不可报名，展商不可自行恢复该状态。
